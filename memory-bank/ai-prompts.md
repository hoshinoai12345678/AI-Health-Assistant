# AI大健康助手 - AI 提示词模板

## 📋 文档说明

本文档包含在开发过程中使用的各类 AI 提示词模板，帮助您高效地与 AI 协作完成开发任务。

---

## 🚀 项目启动提示词

### 1. 首次对话 - 需求澄清

```
你好！我正在开发一个 AI 大健康助手项目（微信小程序）。

请先阅读 memory-bank/ 目录下的所有文档：
- product-requirements.md（产品需求文档）
- tech-stack.md（技术栈文档）
- implementation-plan-phase1.md（实施计划第一阶段）
- architecture.md（系统架构文档）

阅读完成后，请回答以下问题：

1. 实施计划是否完全清晰？有哪些地方需要我进一步澄清？
2. 技术栈选择是否合理？有没有更好的建议？
3. 有哪些潜在的技术风险需要提前考虑？
4. 数据库设计是否合理？有没有遗漏的表或字段？
5. 系统架构是否存在瓶颈或单点故障？

请逐一列出你的问题和建议，我会一一解答。
```

## 🛠️ 开发阶段提示词

### 2. 开始第一步开发

```

请按照以下要求执行：

3. 创建 backend/requirements.txt 文件
4. 创建 backend/app/main.py 文件（基础 FastAPI 应用）
5. 创建 backend/app/core/config.py 文件（配置管理）
6. 创建 .gitignore 文件
7. 创建 README.md 文件

**代码规范**：
- 单文件不超过 200 行
- 添加必要的注释
- 使用类型提示（Type Hints）
- 遵循 PEP 8 规范

**验证标准**：
- [ ] 项目目录结构创建完成
- [ ] Git 仓库初始化成功
- [ ] 后端项目可以启动：uvicorn app.main:app --reload

- 我会负责运行测试验证

- 完成后我会更新 progress.md 和 architecture.md

```

## 📋 总结与提醒

### 使用这些提示词的最佳实践：

1. **明确目标**：清楚地说明你要做什么
2. **提供上下文**：让 AI 阅读相关文档
3. **具体要求**：列出具体的技术要求和验证标准
4. **分步执行**：一次只做一个步骤
5. **及时验证**：完成后立即测试验证
6. **记录进度**：更新 progress.md

### 与 AI 协作的原则：

- ✅ 提供充分的上下文信息
- ✅ 明确说明期望的输出
- ✅ 设定清晰的验收标准
- ✅ 及时反馈和纠正
- ✅ 保持对话的连贯性

### 避免的做法：

- ❌ 模糊的需求描述
- ❌ 一次性要求太多功能
- ❌ 跳过测试验证
- ❌ 不提供错误信息
- ❌ 频繁切换上下文
